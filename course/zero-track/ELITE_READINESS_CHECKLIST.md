# Checklist de prontidão Elite — Trilha Zero

## Conteúdo

- [ ] Z00–Z09 possuem objetivos observáveis.
- [ ] Pré-requisitos estão explícitos.
- [ ] Cada unidade possui prática guiada e independente.
- [ ] Erros comuns e stop conditions estão documentados.
- [ ] Evidência mínima e critérios de conclusão estão definidos.
- [ ] Links e frontmatter passam no CI.

## Execução

- [ ] Exemplos executam em Python 3.11+.
- [ ] Self-tests cobrem sucesso e falhas.
- [ ] Nenhum exemplo exige chave, conta paga ou rede.
- [ ] Entradas desconhecidas são rejeitadas quando aplicável.
- [ ] Operações de escrita demonstram idempotência quando aplicável.

## Segurança

- [ ] Nenhuma credencial real está presente.
- [ ] Logs redigem dados sensíveis.
- [ ] Exemplos não leem arquivos pessoais ou variáveis de ambiente sem necessidade explícita.
- [ ] Comandos destrutivos possuem contenção e aviso.
- [ ] Segurança é critério de bloqueio na avaliação.

## Didática

- [ ] Linguagem simples não elimina precisão técnica.
- [ ] Jargão é definido antes do uso.
- [ ] Cada conceito possui demonstração.
- [ ] A pessoa consegue explicar, não apenas copiar.
- [ ] Soluções comentadas revelam raciocínio e limitações.

## Acessibilidade

- [ ] Estrutura de títulos é consistente.
- [ ] Tabelas possuem leitura compreensível.
- [ ] Diagramas têm alternativa textual.
- [ ] Comandos não dependem apenas de cor.
- [ ] Há alternativas para Windows, Linux e macOS quando necessário.
- [ ] Barreiras encontradas no piloto são registradas.

## Evidência pedagógica

- [ ] Diagnóstico inicial aplicado.
- [ ] Piloto executado com perfis diversos.
- [ ] Tempo, abandono, ajuda e erros foram medidos.
- [ ] Avaliação final foi aplicada.
- [ ] Limitações foram publicadas.
- [ ] Revisões pós-piloto foram concluídas.

## Gate de comunicação

Só usar a expressão “adequado para iniciantes absolutos” quando todos os itens de evidência pedagógica estiverem concluídos e revisados. Antes disso, usar “desenvolvido para iniciantes e em validação pedagógica”.
