# 00 — Orientação e método de engenharia

## Objetivos

- Navegar a arquitetura da NEXUS e explicar por que conceitos e adapters são separados.
- Executar o validador local e registrar uma decisão arquitetural pequena.
- Aplicar o ciclo hipótese–experimento–evidência–revisão.

## Pré-requisitos

Git e terminal em nível introdutório; Python 3.11+ recomendado para os validadores.

## Progressão NEXUS

Conceito: engenharia baseada em evidência → arquitetura: camadas do repositório → implementação: ambiente local →
comparação: tutorial versus laboratório → projeto real: ADR de ambiente reproduzível.

## Laboratórios

- [LAB-000](../../../labs/LAB-000-repository-orientation.md) — validar repositório e rastrear um conceito.

## Projeto

Entregue um ADR com escolha de ambiente, riscos, alternativa rejeitada e saída do validador.

## Checklist

- [ ] Sei localizar conceito, adapter, laboratório e projeto.
- [ ] Meu ADR contém consequência e critério de revisão.
- [ ] Nenhum segredo entrou no histórico Git.
