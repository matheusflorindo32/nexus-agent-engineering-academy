# 11 — Capstone production-grade

## Objetivos

- Integrar arquitetura, contexto, tools, loops, avaliação, SRE e segurança.
- Defender decisões com evidência, ADRs e comparação multiplataforma.
- Operar um piloto controlado com rollback e postmortem.

## Pré-requisitos

Módulos 00–10 concluídos e projeto aprovado no [brief do capstone](../../../projects/capstone/README.md).

## Progressão NEXUS

Conceito: sistema sociotécnico → arquitetura: C4 + threats + SLOs → implementação: vertical slice → comparação: adapter
alternativo → projeto real: piloto controlado e defesa técnica.

## Laboratórios

- LAB-1101 — game day: dependência falha, injeção aparece e budget é consumido.

## Projeto

Entregar sistema executável, demo segura, eval report, threat model, runbook, ADRs, custos, limitações e roadmap.

## Checklist

- [ ] Requisitos e não objetivos são rastreáveis.
- [ ] Eval e segurança bloqueiam release quando falham.
- [ ] Operador consegue observar, pausar, retomar e reverter.
- [ ] Outra pessoa reproduz a demo sem segredo compartilhado.

## Critérios de excelência

- A entrega integra avaliação, segurança, operação e rollback com evidência suficiente para reprodução independente.
