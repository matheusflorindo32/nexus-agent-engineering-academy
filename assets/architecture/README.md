# Arquitetura visual NEXUS

Este diretório reúne os mapas oficiais do sistema NEXUS Agent Engineering Academy.

## Hierarquia de uso

### 1. `nexus-system-light-3d.png`

Asset visual principal e canônico.

Usos aprovados:

- README principal;
- abertura do curso;
- página inicial da documentação;
- apresentação institucional;
- PDFs e materiais de divulgação;
- visão geral antes dos módulos técnicos.

Requisitos antes da publicação final:

- conferir todos os textos em 100% de zoom;
- validar nomes contra a arquitetura real do repositório;
- gerar versão WebP otimizada;
- gerar versão responsiva para telas pequenas;
- manter texto alternativo multilíngue;
- não alterar a paleta clara sem revisão de design.

### 2. `nexus-system-handwritten.png`

Asset pedagógico secundário.

Usos aprovados:

- Módulo 00;
- explicação de design thinking;
- bastidores da construção;
- comparação entre rascunho e arquitetura final;
- vídeos, aulas e publicações educacionais.

Não deve substituir o infográfico 3D claro no README principal.

## Derivados planejados

| Derivado | Dimensão | Uso |
|---|---:|---|
| README desktop | 1600 × 900 | GitHub e documentação |
| Social vertical | 1080 × 1350 | LinkedIn e Instagram |
| Vídeo horizontal | 1920 × 1080 | YouTube e aulas |
| Story | 1080 × 1920 | Stories e Shorts |
| PDF | A4 horizontal | material institucional |
| Mobile simplified | 1080 × 1920 | leitura em telas pequenas |

## Controle de qualidade

Nenhum asset entra em publicação estável sem:

- proveniência registrada;
- licença definida;
- revisão tipográfica;
- texto alternativo;
- correspondência com o conteúdo técnico;
- aprovação humana;
- registro no `assets/ASSET_MANIFEST.yml`.
