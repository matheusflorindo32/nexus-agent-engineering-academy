# Públicos, diagnóstico e pré-requisitos reais

## Princípio

Pré-requisito deve refletir o que a pessoa realmente precisa fazer, não apenas o que o texto afirma.

## Perfis de entrada

### Perfil A — iniciante absoluto

Ainda não consegue, sem ajuda:

- abrir e usar terminal;
- instalar ou localizar Python;
- clonar um repositório;
- interpretar erro simples;
- editar Markdown, JSON ou YAML;
- diferenciar arquivo, pasta, comando e programa.

Entrada recomendada: **Trilha Zero completa**.

### Perfil B — iniciante técnico

Consegue:

- navegar por pastas;
- usar comandos básicos;
- executar Python;
- clonar ou baixar projeto;
- editar arquivos;
- ler código simples.

Entrada recomendada: diagnóstico e possível início no Módulo 00.

### Perfil C — intermediário

Já programa e compreende:

- funções;
- tipos de dados;
- testes básicos;
- Git;
- HTTP e APIs em nível introdutório.

Entrada recomendada: Módulo 00 ou aceleração até o Módulo 02, conforme diagnóstico.

### Perfil D — avançado

Já atua com software, dados, automação ou IA e domina práticas de desenvolvimento.

Entrada recomendada: diagnóstico, módulos obrigatórios de segurança e avaliação, depois trilha especializada.

## Diagnóstico de entrada

A pessoa deve demonstrar as seguintes tarefas em ambiente local:

1. identificar sistema operacional e versão do Python;
2. abrir o terminal na pasta correta;
3. clonar ou baixar o repositório;
4. executar um script Python simples;
5. interpretar sucesso e falha;
6. editar um arquivo Markdown;
7. reconhecer JSON e YAML;
8. explicar por que uma chave de API não deve ir para o Git;
9. executar um teste;
10. registrar o commit analisado.

## Regra de encaminhamento

| Resultado | Encaminhamento |
|---|---|
| 0–3 tarefas | Trilha Zero completa |
| 4–6 tarefas | Trilha Zero acelerada |
| 7–8 tarefas | Módulo 00 com reforço |
| 9–10 tarefas | Módulo 00 ou teste de avanço |

## Pré-requisitos da trilha principal

Antes do Módulo 00, a pessoa deve conseguir:

- usar terminal em nível básico;
- executar Python 3.11+;
- clonar ou baixar projeto;
- editar Markdown;
- reconhecer JSON/YAML;
- executar testes;
- proteger segredos;
- pedir ajuda apresentando comando, erro e contexto.

## Linguagem pública recomendada

Enquanto a Trilha Zero não estiver implementada e validada:

> A NEXUS é recomendada para pessoas com conhecimentos introdutórios de programação, terminal e Git.

Após implementação e piloto bem-sucedido:

> A NEXUS oferece uma entrada guiada para iniciantes absolutos e progressão até engenharia avançada de agentes.
